# Pet Animation & Expansion Design

**Date:** 2026-05-23  
**Status:** Approved

## Problem

Three issues with the current pocket monster pet system:

1. **Direction mapping bug** â€” walking up shows left-facing sprite; walking right/left shows up-facing sprite; only down is correct. Root cause: the PMD `Walk-Anim.png` direction row indices `[0, 4, 6]` extract the wrong rows from the source sprite sheet.
2. **Frame count cap** â€” `Math.min(4, petFrames)` in `pets.ts` limits all species to 4 walk frames even when their PMD sheet has 6 or 8.
3. **Only 5 species** â€” the download script has a hardcoded list of 5 starters; no rotation mechanic.

## Goals

- Fix direction extraction so all 4 directions (down, up, right, left-flipped) render correctly.
- Play the full walk animation frame count per species.
- Expand to ~200 species (all 151 Gen 1 + ~50 fan-favorites from later gens).
- Spawn exactly 5 random pets on startup; replace them with 5 new random ones every 10 minutes using the existing matrix despawn/spawn effects.

## Architecture

No new files. Changes touch: `download-starter-pets.ts`, `convert-pmd-sprite.ts`, `pets.ts`, `officeState.ts`, `constants.ts` (webview), `types.ts` (webview â€” Pet interface), `renderer.ts` (webview).

## Direction Fix

### Investigation approach

During implementation, add a `--debug-dirs` flag to `convert-pmd-sprite.ts` that writes each extracted PMD direction row to a separate `<speciesId>_dir<N>.png` file for visual inspection. Run against one species, identify which PMD indices correspond to Down/Up/Right, update `PMD_DIRS_TO_EXTRACT` in both scripts.

### Expected outcome

`PMD_DIRS_TO_EXTRACT` in both `download-starter-pets.ts` and `convert-pmd-sprite.ts` are updated to extract the correct rows. The 3-row output PNG layout stays the same (`[down, up, right]`) â€” only the source row indices change. All existing 5 pet PNGs are re-generated by re-running the download script.

### Left direction

Left is still derived at load time by `flipHorizontal` on the right frames. No PNG format change needed.

## Frame Count Fix

Remove `Math.min(4, petFrames)` from `pets.ts`. The frame counter advances modulo the actual frame count reported by `getPetSprites(pet.speciesId)?.[pet.dir]?.length`. Species with 4, 6, or 8 walk frames all animate correctly.

## Species Expansion

### Download script

`download-starter-pets.ts` gets a new `SPECIES` array with ~200 entries:

- All 151 Gen 1 species (PokĂ©dex numbers `0001`â€“`0151`)
- ~50 fan-favorites from later gens: Togepi, Marill, Pichu, Cleffa, Igglybuff; Totodile, Cyndaquil, Chikorita, Espeon, Umbreon, Umbreon evos; Torchic, Mudkip, Treecko; Lucario, Garchomp, Gardevoir, Mimikyu, Eevee evolutions (Vaporeon through Sylveon), Gengar, Snorlax, Mewtwo, Mew, Celebi, Jirachi, Manaphy, and other widely recognized species with confirmed PMDCollab walk sprites.

Script is **idempotent**: if `<speciesId>.png` already exists in `assets/pets/`, that species is skipped. Failed fetches (HTTP error or missing Walk-Anim) are logged and skipped â€” they don't abort the run.

### Committed assets

All successfully downloaded PNGs are committed to `webview-ui/public/assets/pets/`. The `.gitkeep` placeholder remains.

## Pet Rotation

### Constants

Replace `PET_COUNT_MIN` and `PET_COUNT_MAX` in `webview-ui/src/constants.ts` with:

```ts
export const PET_SPAWN_COUNT = 5;
export const PET_ROTATION_INTERVAL_SEC = 600; // 10 minutes
```

### OfficeState changes

`OfficeState` gains one new field:

```ts
private petRotationTimer = PET_ROTATION_INTERVAL_SEC;
```

A new method `tickPetRotation(dt: number)` is called from `gameLoop.ts` on each rAF tick, alongside the existing `updatePet` calls:

1. Decrement `petRotationTimer` by `dt`.
2. When `petRotationTimer <= 0`: trigger despawn on all current pets (set `matrixEffect: 'despawn'` on each), schedule a one-shot callback after `MATRIX_EFFECT_DURATION` to clear the pets map and call `spawnPets(getLoadedPetSpecies())`, then reset `petRotationTimer = PET_ROTATION_INTERVAL_SEC`.

The one-shot callback is implemented as a `pendingPetRespawn` field (a countdown from `MATRIX_EFFECT_DURATION`) also ticked each frame â€” no `setTimeout`, keeping all state in the game loop.

### spawnPets update

Replace `PET_COUNT_MIN + Math.floor(Math.random() * (PET_COUNT_MAX - PET_COUNT_MIN + 1))` with the fixed `PET_SPAWN_COUNT = 5`. Species selection: shuffle loaded species, take first 5. If fewer than 5 species are loaded, spawn however many exist.

### Despawn on pets

The `Pet` interface in `types.ts` gains `matrixEffect: 'spawn' | 'despawn' | null` (same shape as `Character.matrixEffect`). `renderer.ts` calls the existing `renderMatrixEffect` function in place of the normal cached-sprite draw when a pet has an active effect â€” `renderMatrixEffect` already works off the sprite's actual pixel dimensions so no size changes are needed. During the effect the pet's FSM is paused (no position or frame updates), matching the character behaviour. Pets mid-despawn are skipped in the renderer's hit-testing path.

## Error Handling

- Download failures: logged, skipped, never throw.
- Fewer than 5 loaded species: spawn however many are available (same as current `spawnPets`).
- No walkable tiles: `spawnPets` already guards this; no change.
- Pet rotation fires even if no agents are present â€” pets are independent of agent state.
